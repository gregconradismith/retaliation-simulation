Model notes

Retaliation Simulation

The current model is a scaffold for building the game. It should be treated as a toy dynamical system until the intended rules are specified.

Current state variables

Each agent has a grievance level between zero and one. External provocations raise grievance. Agents above a response threshold can transmit part of their grievance to a neighbor. Repair damps grievance at every step.

What is established here

The implementation is only a browser-ready simulation scaffold: parameter controls, canvas visualization, status metrics, offline caching, and GitHub Pages deployment. No empirical or theoretical claim about retaliation is implied by the placeholder equations.

Likely next decisions

  • Define the player objective and scoring rule.
  • Specify whether agents should represent individuals, groups, or abstract states.
  • Choose deterministic, stochastic, networked, or game-theoretic update rules.
  • Decide what feedback should be visible before and after each player action.
